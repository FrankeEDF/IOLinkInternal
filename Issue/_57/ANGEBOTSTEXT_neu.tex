Nach aufwändiger Analyse der V04-Spezifikation wurde festgestellt, dass
\textbf{14 von 15 blau markierten Anforderungen umgesetzt werden
können}. Die Anforderung (ACCESS Bytes 0x08) basiert auf einem

\hypertarget{analyse-und-kluxe4rung-bereits-erfolgt}{%
\subsubsection{1. Analyse und Klärung (bereits
erfolgt)}\label{analyse-und-kluxe4rung-bereits-erfolgt}}

\begin{itemize}
\tightlist
\item
  Vollständige Analyse der V04-Spezifikation
\item
  \textbf{Extraktion und Dokumentation aller 15 blau markierten
  Änderungen}
\item
  \textbf{Systematischer Abgleich jeder Anforderung:}

  \begin{itemize}
  \tightlist
  \item
    FB0: LED OFF Verhalten
  \item
    FB1: LED grün, Default beim Start
  \item
    FB2: LED grün, Key A/B Defaults, LED Offline, ACCESS Bytes
  \item
    FB3: Baudrate-Sperre, \textbf{Klärung dass zyklische Telegramme im
    Tunnel Mode zwingend verboten sind} (sporadische Telegramme
    stören/verhindern Funktion)
  \item
    Systemneustart: Alle Default-Werte
  \end{itemize}
\item
  Tiefenanalyse der Firmware-Architektur:

  \begin{itemize}
  \tightlist
  \item
    Register-Handling und Trigger-Logik
  \item
    RFID-Authentifizierung und Block-Schreiben
  \item
    LED-Steuerung und Funktionsbausteinwechsel
  \item
    Default-Werte bei Systemneustart
  \item
    Modbus-Dokumentation
  \end{itemize}
\item
  \textbf{Identifikation des Missverständnisses bzgl. ACCESS Bytes
  (0x08)}
\item
  Multiple Iterationen mit Rückfragen und Korrekturen im wesentlichen
  über GitHub
\item
  Klärung über die tatsächliche Implementierung
\end{itemize}

\textbf{Ergebnis:}

\begin{itemize}
\tightlist
\item
  \textbf{14 Anforderungen spezifiziert und dokumentiert}
\item
  \textbf{1 Anforderung (ACCESS Bytes 0x08) als nicht erforderlich
  identifiziert:}

  \begin{itemize}
  \tightlist
  \item
    Die Firmware muss Key A/B NICHT zu einem Datenblock zusammenbauen
  \item
    ACCESS Bytes funktionieren bereits über Register 1018-1025
    (identisch zu allen anderen Blöcken)
  \item
    Separate Register sind weder nötig noch sinnvoll
  \end{itemize}
\item
  \textbf{Wichtige Klärung zu FB3 (Tunnel Mode):}

  \begin{itemize}
  \tightlist
  \item
    Zyklische und Einzel-Senden-Telegramme MÜSSEN zwingend abgeschaltet
    werden
  \item
    Sporadische automatische Telegramme stören/verhindern die
    einwandfreie Funktion im Tunnel Mode
  \item
    Diese Klärung ist essenziell für die korrekte Implementierung
  \end{itemize}
\end{itemize}
